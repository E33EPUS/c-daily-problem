# C语言每日一题

> 本插件由 E33EPUS 使用 Claude Code 辅助开发

为 MaiBot 设计的 C 语言程序阅读题插件。用户通过指令获取一道给出代码段的题目，猜测程序输出结果，答完后查看答案和解析。题目由 LLM 动态生成，内置题库作为兜底，不限制每日做题次数。

## 功能特性

- LLM 动态出题：优先调用 Bot 绑定的 LLM 模型生成新题，风格参考 LeetCode、牛客网、PTA 的程序阅读题型，知识点覆盖面广
- 题库兜底：LLM 不可用（网络异常、模型未配置等）时自动回退到内置题库，保证功能始终可用
- 不限制次数：提供 `/每日一题` 和 `/再来一题` 两条指令，无需等待次日刷新
- 群组隔离：每个群的题目、答题状态独立存储，互不影响
- 答案匹配宽松：忽略首尾标点、多余空格、引号包裹等差异，降低格式原因导致的误判
- 安全输出：消息中的尖括号等特殊字符会被转义，避免 QQ 平台意外解析

## 指令说明

| 指令 | 说明 | 示例 |
|------|------|------|
| `/每日一题` | 获取一道 C 语言阅读题 | `/每日一题` |
| `/再来一题` | 换一道新题 | `/再来一题` |
| `/答 <答案>` | 提交答案，提示正确或错误 | `/答 11` |
| `/答案` | 查看当前题目的正确答案和解析 | `/答案` |
| `/题解` | 查看当前题目的详细解析 | `/题解` |

指令前缀支持 `/` 和 `!`（即 `!每日一题` 同样有效）。

## 配置说明

配置文件位于 `config.toml`，亦可在 MaiBot WebUI 中修改。

### `[plugin]`

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | `true` | 是否启用插件 |
| `config_version` | str | `"2.0.0"` | 配置版本，由插件自动管理，无需手动修改 |

### `[daily_problem]`

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | `true` | 是否启用编程一题功能 |
| `llm_first` | bool | `true` | 优先使用 LLM 生成题目。关闭后仅从内置题库抽取 |
| `reveal_on_wrong` | bool | `false` | 答错后是否立即显示正确答案和解析。开启后答错可见完整解析；关闭后需通过 `/答案` 或 `/题解` 查看 |

### 配置示例

```toml
[plugin]
enabled = true
config_version = "2.0.0"

[daily_problem]
enabled = true
reveal_on_wrong = false
llm_first = true
```

## 题目来源

### LLM 生成（默认优先）

插件通过 `self.ctx.llm.generate()` 调用 Bot 绑定的 LLM 模型，使用结构化 prompt 要求模型生成 JSON 格式的题目数据（代码、问题、答案、提示、解析）。生成失败时静默回退到题库，不影响用户体验。

LLM 生成的题目每次不同，能够覆盖更广泛的知识点和代码风格。使用的提示词要求模型避开 i++/++i、&& 短路、逗号运算符、switch 穿透等过于常见的基础题型，优先生成指针运算、位操作、static 变量、宏展开陷阱、结构体对齐、大小端、未定义行为等更有区分度的内容。

### 内置题库（兜底）

内置 43 道精选题目，覆盖以下知识点：

- 运算符优先级（算术、位运算、关系运算的优先级差异）
- 自增/自减（i++ 与 ++i 的区别，序列点与未定义行为）
- 指针与数组（指针算术、`2[a]` 等价写法、`sizeof` 数组与指针的区别）
- 位运算（按位与/或/异或/取反/移位）
- 逻辑短路（`&&` 和 `||` 的短路求值特性）
- 逗号运算符（依序求值、返回最右值）
- switch 穿透（无 break 时的 fall-through 行为）
- 类型转换（强制截断、char 与 int 互转）
- 静态变量（`static` 的生命周期和初始化特性）
- 宏展开（纯文本替换导致的优先级陷阱）
- printf 返回值（输出字符数）
- 循环控制（continue 跳过、break 跳出）
- 递归（阶乘等经典递归函数）
- 字符串（`strlen` 遇到 `\0` 停止）
- 结构体对齐（内存 padding 对 `sizeof` 的影响）
- 浮点精度（二进制浮点无法精确表示十进制小数）
- 大小端（union 检测系统字节序）
- 枚举（枚举值自动递增规则）
- 内存管理（`malloc`/`free`、悬空指针、use-after-free）
- 函数指针（动态切换被调用函数）

每个群独立追踪已出过的题库题目，避免短期内重复。

## 安装

### 方式一：插件市场安装（推荐）

在 MaiBot WebUI → 插件市场中搜索 `c-daily-problem`，点击安装即可。插件将自动下载到 `plugins/` 目录并加载。

### 方式二：手动克隆

```bash
git clone https://github.com/E33EPUS/c-daily-problem.git plugins/c-daily-problem
```

克隆后重启 MaiBot，插件将自动加载。默认配置会自动生成。

## 常见问题

**Q: 发 `/每日一题` 没反应？**
A: 确认 Bot 在线、插件已启用（`[plugin].enabled = true`）。检查日志中是否有 `daily-problem` 相关错误。

**Q: 题目一直出题库的，没有 LLM 生成？**
A: 检查 `[daily_problem].llm_first` 是否为 `true`，并确认 Bot 已配置有效的 LLM 模型。LLM 生成失败不会报错，会静默回退到题库。可查看日志中的 debug 信息确认原因。

**Q: 答对了但是判错了？**
A: 答案匹配会做标准化处理（去标点、去引号、去多余空格）。如果仍误判，可能是题目的预期答案格式特殊，欢迎提 Issue。

**Q: 状态文件在哪？**
A: 插件目录下的 `daily_state.json`，存储各群的已出题列表和当前题目缓存。删除此文件可重置所有群的题目状态。
